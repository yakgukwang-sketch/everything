# AI 에이전트 활용 전략

## 핵심 비유: 안드로이드 OS

우리는 에이전트를 만들지 않는다. 에이전트가 사업할 수 있는 **운영체제**를 만든다.

안드로이드가 앱 개발자에게 OS(카메라 API, GPS API, 결제 API)를 제공하듯, 우리는 에이전트에게 세 가지 인프라를 제공한다:

1. **데이터 API** — 전국 로컬 서비스 데이터
2. **결제 처리** — PG + 크립토 통합 결제
3. **웹페이지** — 소비자-에이전트 연결 인터페이스

에이전트는 추천 알고리즘에만 집중하면 된다. 데이터 수집도, 결제 구축도, UI 개발도 필요 없다.

---

## 에이전트 생태계

에이전트는 각각 **영업사원**이다. 실제 시장의 다양한 판매원처럼, 각 에이전트는 고유한 성격과 전략을 가진다.

- **진중한 영업사원** (어드바이저봇): 데이터를 종합 분석, 장단점을 설명하고 신중하게 추천
- **최저가 영업사원** (최저가봇): "이거 사세요. 가장 쌉니다. 끝." 단도직입
- **할인 전문** (알뜰봇): 특가, 무료배송, 쿠폰 조합으로 가장 알뜰한 딜
- **감성 영업사원** (선물봇): "받는 분이 정말 좋아하실 거예요!" 감성 소구
- **트렌드 세터** (트렌드봇): "지금 이게 대세입니다" 유행 기반 추천
- **데이터 분석가** (가격예측봇): "지금 사면 3개월 내 최저가입니다" 수치 기반

에이전트들이 서로 경쟁하고, 소비자가 선택하고, 평가가 쌓이면서 자연스러운 시장이 형성된다.

### 에이전트 프로필 & 단골 시스템

소비자는 매번 가격을 비교하지 않는다. 한번 신뢰할 만한 에이전트를 찾으면 그 에이전트에게 맡긴다. 옛날 동네 단골 세일즈맨과 같다.

```
에이전트 A "맛집헌터"
⭐ 4.8 (거래 12,340건) | 수수료 3%
전문: 숨은 맛집 발굴, 리뷰 분석 특화
"이 에이전트 추천은 진짜 다 맛있음" — 최근 리뷰

에이전트 B "알뜰배달"
⭐ 4.5 (거래 28,900건) | 수수료 2%
전문: 최저가 추천, 크립토 결제 할인
"싸고 빠름. 더 말할 게 없음" — 최근 리뷰
```

소비자가 에이전트를 선택하면, 그 다음부터는 "점심 시켜줘"라고만 하면 에이전트가 알아서 처리한다.

### 결제 인프라: 에이전트가 결제를 직접 안 해도 됨

결제는 우리 플랫폼이 통합 처리한다. 에이전트가 결제 시스템까지 각자 구축하면 진입장벽이 높아진다. 알고리즘 하나 잘 만든 개발자가 에이전트 사업을 바로 시작할 수 있어야 한다.

**크립토 vs 카드 가격 차이:**

| 결제 수단 | 10,000원 주문 | 수수료 |
|-----------|--------------|--------|
| 카드 결제 | 10,300원 | PG 수수료 ~3% |
| 크립토 결제 | 10,000원 | ~0% |

에이전트가 "크립토 결제 시 300원 할인"을 내세워 경쟁할 수 있다. 결제 수단이 에이전트의 차별화 포인트가 된다.

### 사전 질문 정제 (우리의 역할)

소비자와 에이전트 사이의 **통신 규격을 표준화**하는 인프라 역할. 에이전트마다 각각 질문하면 소비자가 피곤해지므로, 사전 질문은 우리가 담당한다.

```
소비자: "침대 사고싶어"
시스템: 어떤 사이즈? → [싱글/퀸/킹]
시스템: 매트리스 종류? → [메모리폼/스프링/라텍스]
시스템: 예산은? → [30만/50만/100만]
→ 조건 정리 후 12개 에이전트에게 동시 전달
→ 에이전트들이 각자 전략으로 경쟁 응답
```

## 배달 서비스 시나리오 (구체화)

### 소비자 요청

> "12시까지 4명 4만원이하로 맞춰서 점심 시켜놔. 남자 2명 여자 2명"

### 사전 질문 (우리 웹페이지에서 처리)

- 못 먹는 음식이 있나요? → [없음] [해산물] [돼지고기] [기타]
- 알러지가 있나요? → [없음] [땅콩] [유제품] [기타]
- 같이 먹어도 되나요? → [같이 먹기] [각자 따로]
- 선호 장르? → [한식] [중식] [양식] [일식] [상관없음]

### JSON 신호 전송

```json
{
  "deadline": "12:00",
  "headcount": 4,
  "budget": 40000,
  "group": { "male": 2, "female": 2 },
  "restrictions": ["none"],
  "allergies": ["none"],
  "sharing": "together",
  "preference": "any",
  "location": "강남역 2번출구 근처",
  "payment_preference": "any"
}
```

### 에이전트들의 경쟁 응답

**에이전트 A (최저가형)**
- 39,500원 | 수수료 2%
- "동네 한식당 제육볶음 세트, 4인분에 딱 맞음"
- 배달비 2,000원 포함
- 💳 카드: 39,500원 / 🪙 크립토: 38,300원

**에이전트 B (취향분석형)**
- 38,000원 | 수수료 4%
- "양식 추천. 이유: 당신이 지난주에 파스타 주문 후 별점 5점 줬음"
- 보험 포함
- 💳 카드: 38,000원 / 🪙 크립토: 36,900원

**에이전트 C (숨은맛집형)**
- 37,000원 | 수수료 3%
- "이 동네 숨은 맛집. 리뷰 127개 평균 4.8"
- 식당 협찬 할인 적용
- 💳 카드: 37,000원 / 🪙 크립토: 35,900원

### 이후 과정 (전부 에이전트 자율)

음식점 주문, 라이더 매칭, 결제, 보험 — 모든 과정을 에이전트가 알아서 처리한다. 단, **결제는 우리 인프라**를 통해 처리되므로 에이전트는 결제 시스템을 직접 구축할 필요가 없다.

라이더 매칭도 별도 에이전트들의 경쟁:

```
배달에이전트 X: 1.2km | 3,500원 | 한집배달 O
배달에이전트 Y: 1.2km | 3,000원 | 한집배달 X (묶음배달)
배달에이전트 Z: 1.2km | 3,200원 | 한집배달 O | 보험포함
```

### 왜 배민보다 싸나?

기존 배민 수수료 20~30%가 사라진다. 그중 절반은 음식점 마진, 나머지 절반은 소비자 할인. 같은 음식이 **약 10% 싸다.** 배민은 수수료를 낮추면 수익 모델이 무너지고, 안 낮추면 소비자가 이탈한다.

## 1단계: 데이터 에이전트 (우리가 만드는 내부 에이전트)

크롤링 자체를 AI 에이전트가 수행한다.

### 자동 크롤링 에이전트

- 사이트 구조가 변경되면 AI가 자동으로 파싱 로직을 재생성
- 새로운 핫딜 소스(어미새 등)를 발견하면 자동으로 크롤러 생성
- 중복 제거: 같은 상품이 여러 커뮤니티에 올라왔을 때 통합
- 가격 정규화: 다양한 표기를 표준 형식으로 변환

### 데이터 품질 에이전트

- 만료된 딜 자동 감지 및 제거
- 허위 딜 필터링 (광고성 게시물, 어뷰징 감지)
- 리뷰/댓글 분석으로 실제 만족도 평가
- 가격 히스토리 추적: "이게 진짜 최저가인가?" 판별

## 2단계: 추천 에이전트 (외부 에이전트가 API를 호출)

우리의 API를 사용하는 외부 AI 에이전트들이 등장한다.

### LLM 기반 에이전트

현재는 규칙 기반(SQL 정렬/필터) 전략 엔진. 다음 단계에서 각 에이전트에 LLM을 연결하면:

- 각 에이전트에 고유 **시스템 프롬프트**(성격)를 부여
- 실제 "생각"해서 추천하는 진짜 AI 영업사원
- 대화 맥락을 이해하고 자연스러운 추천 가능

### 에이전트 유형 예시

| 유형 | 설명 | 수익 모델 |
|------|------|-----------|
| 가격 분석 | 최저가 여부 판단, 구매 타이밍 조언 | API 호출당 과금 |
| 맞춤 알림 | 조건 매칭 시 자동 알림 | 월 구독 / 알림당 |
| 비교 분석 | 쿠폰/적립금/카드 할인 계산한 실구매가 | 분석 건당 |
| 구매 대행 | 할인 상품 발견 → 자동 구매 | 구매 건당 / 절약액 % |

## 3단계: 에이전트 마켓플레이스

```
소비자 → everything 웹 → 에이전트들이 경쟁 → 소비자가 선택
```

소비자가 "닌텐도 스위치 최저가 알려줘"라고 검색하면:

- **에이전트 A**: "쿠팡 349,000원이 최저가. 내일 더 떨어질 확률 12%"
- **에이전트 B**: "네이버 352,000원이지만 네이버페이 3% 적립 → 실구매가 341,440원"
- **에이전트 C**: "지금 사지 마세요. 다음 주 11번가 행사 예상가 320,000원대"

## 4단계: 전체 로컬 서비스로 확장

| 단계 | 데이터 | 에이전트 역할 |
|------|--------|--------------|
| **현재** | 할인 상품 (뽐뿌 → 37개 쇼핑몰) | 최저가 검색, 알림 |
| **다음** | 음식점 (네이버 플레이스, 카카오맵) | 맛집 추천, 주문, 배달 |
| **그 다음** | 숙소 (야놀자, 에어비앤비) | 여행 계획, 최저가 숙소 |
| **최종** | 모든 로컬 서비스 | 생활 전반의 AI 비서 |

## 수익 흐름

```
데이터 수집 → API 제공 → 에이전트 호출 → 결제 처리 → 수익

1. API 호출당 과금 (에이전트 → 우리)
2. 결제 수수료 (거래량 비례)
3. 에이전트 프리미엄 등록비 (검증 배지, 상위 노출)
```

모든 에이전트가 우리 API를 호출하고 우리 결제 시스템을 사용하므로, 에이전트가 누가 이기든 우리는 수익이 난다.

## 에이전트 API 설계 (예정)

```
GET /api/deals?q=에어팟&sort=price_low&limit=10
GET /api/deals?source=ppomppu&category=전자기기
GET /api/deals/:id
GET /api/stats
```

```json
POST /api/agents/register
{
  "name": "최저가봇",

  "endpoint": "https://agent-a.example.com/recommend"
}
```
