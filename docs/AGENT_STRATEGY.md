# AI 에이전트 활용 전략

## 현재 상태

우리는 지금 데이터 수집 인프라를 만들었다. 쇼핑몰(11번가, 다나와)과 커뮤니티(뽐뿌, 루리웹, 클리앙, FM코리아, 퀘사이저존)에서 할인 상품을 크롤링하고 API로 제공한다. 이것은 시작일 뿐이다.

핵심 질문: **이 데이터 위에 에이전트가 어떻게 올라가는가?**

---

## 에이전트 생태계 철학

에이전트는 각각 **영업사원**이다. 실제 시장의 다양한 판매원처럼, 각 에이전트는 고유한 성격과 전략을 가진다.

- **진중한 영업사원** (어드바이저봇): 데이터를 종합 분석, 장단점을 설명하고 신중하게 추천
- **최저가 영업사원** (최저가봇): "이거 사세요. 가장 쌉니다. 끝." 단도직입
- **할인 전문** (알뜰봇): 특가, 무료배송, 쿠폰 조합으로 가장 알뜰한 딜
- **감성 영업사원** (선물봇): "받는 분이 정말 좋아하실 거예요!" 감성 소구
- **트렌드 세터** (트렌드봇): "지금 이게 대세입니다" 유행 기반 추천
- **데이터 분석가** (가격예측봇): "지금 사면 3개월 내 최저가입니다" 수치 기반

이것은 단순한 추천 시스템이 아니라 **하나의 생태계**다. 에이전트들이 서로 경쟁하고, 소비자가 선택하고, 평가가 쌓이면서 자연스러운 시장이 형성된다.

### 사전 질문 정제 (우리의 역할)

소비자와 에이전트 사이의 **통신 규격을 표준화**하는 인프라 역할. 에이전트마다 각각 질문하면 소비자가 피곤해지므로, 사전 질문은 우리가 담당한다.

```
소비자: "침대 사고싶어"
시스템: 어떤 사이즈? → [싱글/퀸/킹]
시스템: 매트리스 종류? → [메모리폼/스프링/라텍스]
시스템: 예산은? → [30만/50만/100만]
→ 조건 정리 후 12개 에이전트에게 동시 전달
→ 에이전트들이 각자 전략으로 경쟁 응답
```

### LLM 기반 에이전트 (다음 단계)

현재는 규칙 기반(SQL 정렬/필터) 전략 엔진이다. 다음 단계에서 각 에이전트에 LLM(Gemini API, Claude API 등)을 연결하면:

- 각 에이전트에 고유 **시스템 프롬프트**(성격)를 부여
- 실제 "생각"해서 추천하는 진짜 AI 영업사원
- 대화 맥락을 이해하고 자연스러운 추천 가능
- 에이전트마다 다른 LLM 모델 사용 가능 (비용/성능 트레이드오프)

## 1단계: 데이터 에이전트 (우리가 만드는 내부 에이전트)

크롤링 자체를 AI 에이전트가 수행한다. 사람이 크롤러를 유지보수하는 것은 한계가 있다. 사이트 구조가 바뀌면 크롤러가 깨지고, 새로운 소스를 추가할 때마다 코드를 짜야 한다.

### 자동 크롤링 에이전트

- 사이트 구조가 변경되면 AI가 자동으로 파싱 로직을 재생성
- 새로운 핫딜 소스(FM코리아, 어미새, 퀘사이저 등)를 발견하면 자동으로 크롤러 생성
- 중복 제거: 같은 상품이 여러 커뮤니티에 올라왔을 때 통합
- 가격 정규화: "12,900원/무배", "12900원(배무)" 등 다양한 표기를 표준 형식으로 변환

### 데이터 품질 에이전트

- 만료된 딜 자동 감지 및 제거 (링크 클릭 → 품절/404 체크)
- 허위 딜 필터링 (광고성 게시물, 어뷰징 감지)
- 리뷰/댓글 분석으로 실제 만족도 평가
- 가격 히스토리 추적: "이게 진짜 최저가인가?" 판별

## 2단계: 추천 에이전트 (외부 에이전트가 API를 호출)

우리의 API를 사용하는 외부 AI 에이전트들이 등장한다. 각 에이전트는 자기만의 전략으로 소비자에게 할인 상품을 추천한다.

### 에이전트 유형 예시

**가격 분석 에이전트**
- "이 상품 지금 사도 되나요?" → 가격 히스토리를 분석해서 최저가 여부 판단
- "3만원 이하 블루투스 이어폰 추천해줘" → 실시간 할인 데이터에서 최적 상품 추출
- 수수료: API 호출당 과금

**맞춤 알림 에이전트**
- 사용자가 관심 카테고리 설정 → 새 딜이 올라오면 즉시 알림
- "에어팟 프로 20만원 이하 되면 알려줘" → 조건 매칭 시 자동 알림
- 수수료: 월 구독 또는 알림당 과금

**비교 분석 에이전트**
- 같은 상품의 여러 소스 가격을 비교
- 쿠폰/적립금/카드 할인까지 계산한 실구매가 산출
- "쿠팡 vs 11번가 vs 네이버 중 어디가 진짜 싼가?" 분석
- 수수료: 분석 건당 과금

**구매 대행 에이전트**
- 할인 상품 발견 → 자동 구매까지 처리
- 타임딜/선착순 상품을 사람보다 빨리 잡아줌
- 수수료: 구매 건당 과금 또는 절약액의 %

## 3단계: 에이전트 마켓플레이스 (우리의 웹에서)

### 구조

```
소비자 → everything 웹 → 에이전트들이 경쟁 → 소비자가 선택
```

소비자가 "닌텐도 스위치 최저가 알려줘"라고 검색하면:

- **에이전트 A**: "현재 쿠팡 골드박스 349,000원이 최저가. 내일 더 떨어질 확률 12%"
- **에이전트 B**: "네이버 스마트스토어 352,000원이지만 네이버페이 3% 적립하면 실구매가 341,440원"
- **에이전트 C**: "지금 사지 마세요. 다음 주 11번가 십일절 행사 예정. 예상가 320,000원대"

소비자가 원하는 에이전트를 선택한다. 에이전트의 추천이 좋으면 평점이 올라가고, 더 많은 소비자가 찾는다.

### 에이전트 등록 시스템

1. 개발자가 API 키 발급
2. 에이전트 프로필 등록 (이름, 전략 설명, 수수료율)
3. 사용자 쿼리에 대해 응답 API 구현
4. 사용자 평가 기반 랭킹

## 4단계: 할인을 넘어서 — 전체 로컬 서비스로 확장

할인 상품 데이터는 시작점이다. 같은 구조를 다른 영역으로 복제한다.

| 단계 | 데이터 | 에이전트 역할 |
|------|--------|--------------|
| **현재** | 할인 상품 (뽐뿌/루리웹/클리앙) | 최저가 검색, 알림 |
| **다음** | 음식점 (네이버 플레이스, 카카오맵) | 맛집 추천, 예약 |
| **그 다음** | 숙소 (야놀자, 에어비앤비) | 여행 계획, 최저가 숙소 |
| **최종** | 모든 로컬 서비스 | 생활 전반의 AI 비서 |

데이터 구조는 동일하다: **이름, 가격, 위치, 리뷰, 시간**. 크롤러와 API 스키마만 확장하면 된다.

## 에이전트 API 설계 (예정)

### 에이전트가 호출하는 API

```
GET /api/deals?q=에어팟&sort=price_low&limit=10
GET /api/deals?source=ppomppu&category=전자기기
GET /api/deals/:id
GET /api/stats
```

### 에이전트가 응답하는 API (마켓플레이스용)

```json
POST /api/agents/register
{
  "name": "최저가봇",
  "description": "실시간 최저가 분석 및 구매 타이밍 추천",
  "commission_rate": 2,
  "endpoint": "https://agent-a.example.com/recommend"
}

POST /api/agents/:id/respond
{
  "query": "닌텐도 스위치 최저가",
  "response": {
    "recommendation": "쿠팡 349,000원 (현재 최저가)",
    "confidence": 0.92,
    "reasoning": "최근 30일 가격 추이 분석 결과..."
  }
}
```

### 에이전트 평가 API

```json
POST /api/agents/:id/review
{
  "rating": 5,
  "comment": "정확한 가격 분석이었음"
}
```

## 수익 흐름

```
크롤링 데이터 → API 제공 → 에이전트 호출 → 소비자 선택 → 수익

1. API 호출당 과금 (에이전트 → 우리)
2. 에이전트 프리미엄 등록비 (검증 배지, 상위 노출)
3. 트래픽 기반 광고 (웹페이지)
```

모든 에이전트가 우리 API를 호출해야 하므로, 에이전트가 많아질수록 수익이 자동으로 따라온다. 에이전트가 누가 이기든 상관없다 — 우리는 원자재를 판다.
