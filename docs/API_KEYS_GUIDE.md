# 크롤링 & 데이터 수집 가이드

## 데이터 수집 전략 (양방향)

### 방향 1: 크롤링

뽐뿌 게시판(ppomppu.co.kr) 하나만 크롤링하여 37개 이상의 쇼핑몰 딜 데이터를 수집한다.

각 쇼핑몰을 직접 크롤링하면 API 키 발급, 차단 대응, 사이트별 파서 유지보수가 필요하지만, 뽐뿌는 사용자들이 이미 각 쇼핑몰에서 검증된 딜을 올려놓은 커뮤니티이므로 하나의 크롤러로 모든 쇼핑몰 데이터를 경제적으로 확보할 수 있다.

### 방향 2: 사업자 직접 등록

음식점, 호텔, 판매자가 직접 데이터를 올리고 판매할 수 있는 기능을 제공한다.

- 여기에 올리면 에이전트 수백 개에 동시에 노출 → 배민/쿠팡이츠 하나에 입점하는 것보다 이득
- 직접 올리면 데이터 정확도 향상 + 크롤링 법적 리스크 감소
- 생태계가 커질수록 직접 등록 비율 증가 → 크롤링 의존도 자연 감소

## 실행 방법

### 최신 딜 수집 (기본)
```bash
cd data
python main.py
```

### 대량 수집
```bash
cd data
python crawl_all.py
```

`crawl_all.py`의 `MAX_POSTS` 값을 조정하여 수집량 제어.

## 수집 데이터

| 필드 | 설명 | 검증 |
|------|------|------|
| title | 게시글 제목 (상품명 + 가격 + 배송비) | 5자 이상 |
| source | 쇼핑몰 (제목의 [출처] 태그에서 추출) | 정규식 |
| price | 가격 (제목에서 숫자 추출) | 0 허용 (미흡) |
| source_id | 뽐뿌 게시글 번호 | 정규식 + 페이지 내 중복 체크 |
| image_url | 썸네일 이미지 | 선택적 |
| recommendations | 추천수 | 선택적 |

## 알려진 제한사항

- **인코딩**: euc-kr 하드코딩 + `errors="replace"` → 일부 문자 유실 가능
- **중복 제거**: 페이지 내만 (크로스 페이지 중복 미처리)
- **재시도**: 없음 — 네트워크 실패 시 해당 페이지 건너뜀
- **로깅**: 전부 `print()` — logging 모듈 미사용
- **업로드 실패**: base.py에서 로컬 JSON 백업 저장 (deals_backup.json)

## API 키 불필요

뽐뿌는 별도 API 키 없이 크롤링 가능. `.env` 파일에 API URL만 설정하면 된다.

```env
API_URL=https://everything-api.deri58.workers.dev
```
